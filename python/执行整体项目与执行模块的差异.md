# IDE与终端执行项目的差异

在使用IDE直接加载项目的情况下，会出现有和终端直接按照`-m`执行`modules`有所不同的行为，因此在个别的情况下，可能会出现编写的代码在IDE中可以执行，但通过终端无法执行的情况。

## 操作系统与文件系统的识别差异

在Windows系统中，文件系统可以识别大小写的差异，其将大小写不同的文件也识别为一个，在我使用`PyCharm`运行项目时，发现即使`import`的文件夹与实际文件夹大小写不同，但似乎`IDE`直接使用了操作系统底层的文件下载机制，如我定义了`User/`目录，但在`app.py`中`import user from User/routes`也可以正常找到。
但是当我进入终端，在我使用
```
python -m app.py
```
试图直接以形式执行整个项目时，大小写不同的`import`直接进行报错，Python 的模块查找机制会更加严格，他无法找到对应的模块，因此进行了报错。

## 虚拟环境的差异

在IDE中，实际上是有一个默认的虚拟环境存在的，在点击运行时会自动加载该虚拟环境中的内容，但如果使用`-m`的模式加载，则是默认的系统环境。

因此，如果在IDE中已经设置了虚拟环境，或者任何存储的以IDE级别的内容，譬如可能将数据库的密码和库名存储在IDE中，使得IDE运行情况下可以直接执行，但如果在终端执行则不可能读取到IDE级别的环境变量，当然，这种情况是在设计过程中本来就应当避免的，任何系统的env内容都应该设置在单独的`.env`中使用`dotenv`来读取，而不是通过IDE配置完成，否则显然是难以部署的。
